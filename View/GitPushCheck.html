Git 推送檢查流程圖

① 確認未提交的變更
使用 git status 檢查哪些檔案已修改、刪除或新增。
git status
→ 顯示目前有哪些檔案準備被提交。
② 檢查與遠端差異
查看與 origin/main 的差異，確認哪些檔案會被推送。
git diff --name-only origin/main
→ 顯示尚未推送的檔案清單。
③ 檢視最後一次提交內容
若已提交但不確定內容，可檢查最後一次 commit。
git log -1 --stat
→ 顯示檔案異動與行數增減統計。
④ 查看遠端分支實際內容
不切換分支也能列出 origin/gh-pages 的檔案清單。
git ls-tree -r origin/gh-pages --name-only
→ 顯示 GitHub Pages 目前的檔案結構。
⑤ 推送更新至遠端
當檢查無誤後，執行推送動作：
git push origin main
或對 GitHub Pages：
git push origin gh-pages
